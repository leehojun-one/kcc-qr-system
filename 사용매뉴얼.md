# KCC 창호 QR 관제 시스템 — 사용 매뉴얼 (초보자용)

> 컴퓨터를 처음 다루는 분도 따라 할 수 있게 **Windows 기준**으로 한 단계씩 설명합니다.
> (맥은 명령어가 거의 같고, 다른 부분은 따로 표시했어요.)

---

## 0. 시작하기 전에 — 이게 뭔가요?

이 프로그램은 **창문에 붙인 QR / 발주서 바코드**로 현장 사고와 고객 A/S를
한 곳에서 관리하는 시스템입니다. 인터넷 사이트가 아니라, **내 컴퓨터에서 켜서**
브라우저(크롬 등)로 보는 방식이에요. 켜는 동안만 작동하고, 끄면 닫힙니다.

준비물은 딱 두 가지입니다.
1. 받은 파일들이 들어있는 폴더 (`kcc_qr_system`)
2. 파이썬(Python) — 프로그램을 돌리는 엔진. (설치법은 1단계)

---

## 1. 파이썬(Python) 설치하기  *(이미 깔려 있으면 건너뛰기)*

1. 인터넷 주소창에 **python.org/downloads** 입력 → 노란 **Download Python** 버튼 클릭.
2. 받은 파일을 실행합니다.
3. ⚠️ **아주 중요** — 설치 첫 화면 맨 아래 **`Add Python to PATH`** 체크박스를 **꼭 체크**하세요.
   (이걸 안 하면 나중에 명령어가 안 먹습니다.)
4. **Install Now** 클릭 → 끝나면 닫기.

> 맥(Mac)은 보통 파이썬이 들어있어요. 1단계는 건너뛰고 바로 2단계로 가도 됩니다.

---

## 2. 폴더 준비하기

1. 받은 `kcc_qr_system` 폴더를 찾기 쉬운 곳(예: **바탕화면**)에 둡니다.
2. 폴더 안에 이런 파일들이 있는지 확인하세요.
   `app.py`, `db.py`, `parser.py`, `notify.py`, `barcode_qr.py`,
   `seed_site.json`, `requirements.txt`, `README.md`

---

## 3. 명령 입력창(터미널) 열기

이 프로그램은 버튼이 아니라 **명령어 한 줄**로 켭니다. 겁먹지 마세요, 복사+붙여넣기면 됩니다.

**가장 쉬운 방법:**
1. `kcc_qr_system` 폴더를 엽니다.
2. 폴더 위쪽 **주소표시줄**(파일 경로가 적힌 칸)을 마우스로 클릭하고,
   글자를 다 지운 뒤 **`cmd`** 라고 입력하고 Enter.
3. 검은 창이 뜨면 성공! 이 창이 폴더 위치에서 열린 **명령 입력창**입니다.

> 맥(Mac): 폴더에서 우클릭 → **"폴더에서 새로운 터미널 열기"**.

---

## 4. 처음 한 번만 — 필요한 부품 설치

검은 창에 아래를 **복사해서 붙여넣고** Enter:

```
pip install -r requirements.txt
```

> 붙여넣기는 검은 창 안에서 **마우스 우클릭**하면 됩니다. (Ctrl+V가 안 될 수 있어요.)

설치가 주르륵 진행되다가 멈추면 완료예요. (1~2분 걸립니다. 처음 한 번만 하면 돼요.)

❓ `pip를 찾을 수 없습니다` 라고 나오면 → 아래 명령으로 대신 해보세요.
```
python -m pip install -r requirements.txt
```

---

## 5. 프로그램 켜기

검은 창에 아래를 붙여넣고 Enter:

```
streamlit run app.py
```

- 잠시 후 **크롬 같은 브라우저가 자동으로 열리면서** 화면이 나타납니다.
- 안 열리면, 검은 창에 적힌 **`Local URL: http://localhost:8501`** 주소를
  복사해서 브라우저 주소창에 붙여넣으세요.

❓ `streamlit을(를) 찾을 수 없습니다` 라고 나오면 → 이렇게 해보세요.
```
python -m streamlit run app.py
```

🎉 화면이 떴다면 성공입니다. 처음 켤 때 예시 데이터가 자동으로 채워져요.

## 6. 화면 사용법

왼쪽에 **화면 선택** 메뉴가 있어요. 세 가지를 눌러가며 씁니다.

### ① 시공팀 앱  (현장 직원용)
- **현장 등록 탭**: 위에서 현장(바코드)을 고르면 발주서 바코드와 QR이 그려집니다.
  `✅ 이 현장에 QR 연결`을 누르면, 그 집 창호가 그 QR에 연결돼요.
  (→ 나중에 고객이 그 QR을 찍으면 이 집 창문 목록이 뜹니다.)
- **사고 / 입주전 AS 등록 탭**:
  - **유형**을 먼저 고릅니다. `당일사고`(설치 당일) 또는 `입주전AS`(공사중·입주 전).
  - 문제 창, 1차 귀책(누구 책임으로 보이는지), 사진을 넣고 접수 버튼.
  - 당일사고는 **가공처+관제**에, 입주전AS는 **관제+영업**에 알림이 갑니다.
    (지금은 실제 카톡 대신 화면에 "이런 메시지가 갑니다" 미리보기로 보여줘요.)

### ② 고객 앱  (입주민용)
- QR 일련번호를 고릅니다. (실제로는 고객이 거실창 QR을 폰으로 찍는 부분)
- **처음이면 정품등록**(연락처·입주일), **그 다음부터는 A/S 접수** 화면이 떠요.
- A/S는 창과 증상을 체크하면 끝. 시공팀·가공처 같은 내부 정보는 고객에게 안 보입니다.

### ③ 관제센터  (본사용)
- 위쪽 숫자판: 하자율, 누적 사고, 미확인 건 등 한눈에.
- **2차 귀책 확정 대기**: 현장이 고른 1차 귀책을 본사가 검토해서 **`확정`** 버튼을 눌러야
  하자율 통계에 반영됩니다. 입주전AS는 **`가공처에 지시 발송`** 버튼으로 가공처에 내려보냅니다.
- 아래쪽: 최근 사고 티켓 목록, 고객 A/S 접수 현황.

## 7. 끄는 법 / 다시 켜는 법

- **끄기**: 검은 창에서 **Ctrl + C** 를 누르면 멈춥니다. 그 다음 창을 닫으면 돼요.
- **다시 켜기**: 3단계(폴더에서 cmd 열기) → 5단계(`streamlit run app.py`)만 반복.
  (4단계 설치는 처음 한 번만 하면 됩니다.)
- **입력한 데이터 싹 지우고 새로 시작**: 폴더 안 **`kcc_qr.db`** 파일을 삭제하고 다시 켜세요.
  (이 파일이 데이터 저장소예요. 켜면 예시 데이터가 다시 깔립니다.)

## 8. 자주 막히는 것 (FAQ)

| 증상 | 해결 |
|---|---|
| `pip / streamlit 을(를) 찾을 수 없습니다` | 1단계에서 **Add Python to PATH** 체크를 빠뜨린 경우. 파이썬을 지우고 그 체크를 켜서 다시 설치하거나, 명령 앞에 `python -m` 을 붙여 실행 (예: `python -m streamlit run app.py`). |
| 브라우저가 자동으로 안 열림 | 검은 창의 `Local URL` 주소를 복사해 브라우저에 붙여넣기. |
| `Port 8501 is already in use` (포트 사용중) | 이미 한 번 켜져 있는 거예요. 다른 검은 창에서 Ctrl+C로 끄거나, `streamlit run app.py --server.port 8600` 처럼 뒤에 다른 번호를 붙여 켜기. |
| 한글이 깨져 보임 | 브라우저 화면은 정상이에요. 검은 창 글자만 깨지는 건 무시해도 됩니다. |
| 화면이 멈춘 것 같음 | 브라우저 오른쪽 위 **⟳(Rerun)** 또는 새로고침(F5). |
| 검은 창을 실수로 닫음 | 프로그램이 꺼진 겁니다. 5단계로 다시 켜면 돼요. (데이터는 안 날아갑니다.) |

## 9. 파일별 역할 (참고용)

| 파일 | 하는 일 | 손대도 되나? |
|---|---|---|
| `app.py` | 화면 전체 (시공팀/고객/관제) | 화면 바꿀 때 |
| `parser.py` | 시공발주서 엑셀 → 데이터 추출 | 양식 바뀔 때만 |
| `db.py` | 데이터 저장소(SQLite) + 예시 데이터 | 본사 서버 옮길 때 |
| `notify.py` | 알림 발송 + 에스컬레이션 규칙 | 카톡 연동할 때 |
| `barcode_qr.py` | 바코드·QR 그림 생성 | 거의 안 건드림 |
| `seed_site.json` | 실제 발주서 1현장 예시 데이터 | — |
| `requirements.txt` | 필요한 부품 목록 | — |
| `kcc_qr.db` | 실제 데이터가 쌓이는 파일 (자동 생성) | 초기화할 때 삭제 |

## 10. 본사 IT에 넘길 때 (요약)

지금은 **내 컴퓨터 데모용**이고, 실제 운영은 본사 서버에서 합니다. IT가 손댈 곳은 3곳:

1. **데이터 저장소**: `db.py` 를 사내 DB로 교체 (화면 코드 `app.py`는 그대로).
2. **알림 실발송**: `notify.py` 의 `_send_via_provider()` 에 카카오 알림톡(솔라피 등) 연결.
   → 알림톡 템플릿은 **사전 심사 1~3일** 필요하니 미리 신청.
3. **하자율 기준**: `db.py` 의 `VENDOR_BASELINE` 를 실제 시공 실적 집계로 교체.

자세한 건 같은 폴더의 **`README.md`** 에 정리돼 있습니다.
