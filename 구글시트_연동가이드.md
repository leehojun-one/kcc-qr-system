# 구글시트 연동 가이드 (초보자용)

AS접수·사고접수·정품등록 기록을 **구글시트에 자동으로 쌓는** 설정입니다.
앱이 재시작돼도 시트에는 기록이 남고, 시트를 열어 바로 볼 수 있어요.
한 번만 설정하면 됩니다. (시각은 자동으로 **한국시간**으로 저장돼요.)

---

## 준비물
- 구글 계정 (회사 또는 개인)
- 약 15분

---

## 1단계 — 구글시트 만들기
1. sheets.google.com 에서 **빈 시트** 하나 새로 만듭니다.
2. 이름 예: `KCC창호_AS로그`.
3. 주소창의 URL에서 **문서 key**를 복사해 둡니다.
   `https://docs.google.com/spreadsheets/d/`**`이부분이_key`**`/edit`
   (`/d/` 와 `/edit` 사이의 긴 문자열)

> 탭(sites·windows·incidents·registrations·as_requests·notif_logs)은 앱이 자동으로 만들어 줍니다.

---

## 2단계 — 서비스 계정 만들기 (앱이 시트에 쓸 '로봇 계정')
1. console.cloud.google.com 접속 → 상단에서 **새 프로젝트** 생성 (이름 예: `kcc-qr`).
2. 검색창에 **Google Sheets API** → 들어가서 **사용(Enable)**.
   (한 번 더: **Google Drive API** 도 검색해서 사용 설정 — 시트 열기에 필요)
3. 왼쪽 메뉴 **API 및 서비스 ▸ 사용자 인증 정보** → **사용자 인증 정보 만들기 ▸ 서비스 계정**.
4. 이름 아무거나(예: `kcc-sheet-bot`) → 만들고 완료.
5. 만든 서비스 계정 클릭 → **키** 탭 → **키 추가 ▸ 새 키 만들기 ▸ JSON** → 파일이 다운로드됩니다.
   이 JSON 파일을 잘 보관하세요. (열어보면 `client_email`, `private_key` 등이 들어있어요.)

---

## 3단계 — 시트를 로봇 계정에 공유 (제일 자주 빠뜨리는 단계!)
1. JSON 파일을 메모장으로 열어 **`client_email`** 값을 복사합니다.
   (`xxxx@kcc-qr.iam.gserviceaccount.com` 형태)
2. 1단계에서 만든 **구글시트** 우측 상단 **공유** 클릭 → 그 이메일을 붙여넣고 **편집자**로 공유.

> 이걸 안 하면 "권한 없음"으로 실패합니다. 꼭 편집자로 공유하세요.

---

## 4단계 — Secrets 에 정보 넣기

앱에게 "이 시트에, 이 로봇 계정으로 써라"라고 알려주는 부분입니다.

### 로컬(내 PC)에서 테스트할 때
앱 폴더 안에 `.streamlit` 폴더를 만들고 그 안에 **`secrets.toml`** 파일을 만들어 아래처럼 채웁니다.

```toml
sheet_key = "여기에_1단계의_key"
base_url  = "https://너의앱주소.streamlit.app/"   # QR용(아직 없으면 빈칸으로 둬도 됨)

[gcp_service_account]
type = "service_account"
project_id = "kcc-qr"
private_key_id = "JSON의 private_key_id 값"
private_key = "-----BEGIN PRIVATE KEY-----\n...JSON의 private_key 그대로...\n-----END PRIVATE KEY-----\n"
client_email = "xxxx@kcc-qr.iam.gserviceaccount.com"
client_id = "JSON의 client_id 값"
auth_uri = "https://accounts.google.com/o/oauth2/auth"
token_uri = "https://oauth2.googleapis.com/token"
auth_provider_x509_cert_url = "https://www.googleapis.com/oauth2/v1/certs"
client_x509_cert_url = "JSON의 client_x509_cert_url 값"
universe_domain = "googleapis.com"
```

> 💡 위 값들은 **다운로드한 JSON 파일을 그대로 옮겨 적는** 거예요.
> 특히 `private_key` 는 JSON에 있는 `\n` 까지 **그대로** 큰따옴표 안에 넣으세요.

### Streamlit Cloud(배포)에서
앱 페이지 우측 하단 **⋮ ▸ Settings ▸ Secrets** 에 **위 내용 그대로** 붙여넣고 Save.
(로컬 secrets.toml 과 똑같은 내용)

---

## 5단계 — 연결 테스트
```
streamlit run test_gsheet.py
```
- "Secrets 설정 ✅" 이 보이고
- **`연결 확인 + 시드 생성`** 버튼 → 초록 성공 메시지 →
- 구글시트에 `sites·windows·incidents` 탭이 생기면 **성공!**

---

## 6단계 — 실제 사용
이제 본 앱(`streamlit run app.py`)을 켜면 **구글시트가 메인 DB**가 됩니다:
- 사이드바에 `🗄️ 저장소: 구글시트 (영구)` 로 표시됨
- 현장/사고/귀책확정/정품등록/AS접수/알림이 모두 시트의 각 탭에 저장·갱신
- Streamlit Cloud가 재시작돼도 **데이터가 그대로 유지**됨
- Secrets 를 빼면 자동으로 로컬 SQLite(임시)로 돌아감

모든 시각은 **한국시간(KST)** 으로 기록됩니다.

---

## 자주 막히는 것
| 증상 | 원인/해결 |
|---|---|
| 로깅이 계속 OFF | Secrets 에 `sheet_key` 또는 `[gcp_service_account]` 가 없음. 4단계 확인 |
| "권한 없음 / PermissionError" | 3단계 — 시트를 서비스 계정 이메일에 **편집자**로 공유 안 함 |
| "API has not been enabled" | 2단계 — Google **Sheets API + Drive API** 사용 설정 안 함 |
| private_key 오류 | `private_key` 줄의 `\n` 을 빼먹었거나 따옴표가 깨짐. JSON 원본 그대로 복사 |

---

## 참고 — 성능/할당량
구글시트는 분당 읽기 횟수 제한이 있어, 앱은 각 탭을 **한 번 읽어 캐시(약 6초)** 하고 메모리에서 처리합니다.
건수가 적은 파일럿에는 충분합니다. 트래픽이 커지면 그때 사내 DB로 옮기면 됩니다.
